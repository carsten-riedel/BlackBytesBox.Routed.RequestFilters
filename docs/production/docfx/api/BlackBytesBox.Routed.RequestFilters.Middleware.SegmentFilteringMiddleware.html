InvokeAsync(HttpContext)

Processes the HTTP request by validating the URI segments of the request URL against the configured whitelist and blacklist patterns. If any segment is blacklisted or the full URI cannot be built, the request is treated as not allowed. If at least one segment is whitelisted, the request is forwarded.

NotMatchedAsync(HttpContext, SegmentFilteringMiddlewareOptions, string)

Asynchronously handles requests for which the URL (displayUrl) does not match any whitelist or blacklist patterns.

Declaration

Parameters
Type Name Description
HttpContext context

The current HTTP context of the request.
SegmentFilteringMiddlewareOptions options

The configuration options for segment filtering middleware.

A string representing the combined URL segments (e.g., "/segment1/segment2/segment3").
Returns
Type Description
Task

A task representing the asynchronous operation.
Remarks

Depending on the options provided, this method logs the event and either continues processing the request or aborts it by writing a default status code response. The displayUrl parameter is used to log the URL segments that were derived from the request path.

Examples
// Example usage:
await NotMatchedAsync(httpContext, middlewareOptions, "/segment1/segment2/segment3");
